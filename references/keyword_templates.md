# Phase A — 关键词扩展模板

按主题 intent 选模板. 复用 `xhs-research-lite` 的搜索矩阵思路.

## 📋 求职 / 公司类 (笔试 / 面经 / 入职 / 离职)

```
1. <公司> + <岗位>            (核心精确)
2. <公司> + 笔试 / 面经
3. <公司> + 面试经验 / 一面 / 二面
4. <公司> + 实习 / 转正 / 入职体验
5. <公司> + 真实感受 / 避坑 / 吐槽
```

**优先全称**: `哔哩哔哩` > `B站`, `中信证券` > `中信`. 全称搜出 OP 比例显著高于缩写.

## 🛠️ 工具 / 产品评测类 (X 工具好用吗 / 替代品)

```
1. <产品名> + 评测 / 体验 / 真实使用
2. <产品名> + vs <主要竞品>
3. <产品名> + 优点 / 缺点 / 缺陷
4. <产品名> + 用法 / 上手 / 教程
5. <产品名> + 不推荐 / 翻车 / 退坑
```

## 💭 人生 / 情感 / 方法论类 (人迷茫 / 怎么办)

**Test 2 验证: 泛词 100% 营销号, 具体年龄+状态出 OP**. 强制 ≥ 2 个具体词:

```
1. <具体年龄> + <具体状态>   (e.g., "25岁 失业", "35岁 裸辞 后悔")
2. <场景> + 经历 / 真实故事
3. <人群> + 反思 / 走出来
4. <场景> + 误区 / 坑
5. <相关情绪> + <人群>
```

❌ 不要泛词: `人迷茫怎么办` / `迷茫 走出来` (营销号红海)
❌ "应届生 / 大学生 / 学生" + 泛词 → 培训机构招生 SEO 必中. 改成"应届生 第一份工作 后悔"类反励志变体.

## 🏢 品牌 / 公司认知类

套 `xhs-research-lite` 默认 4 类入口 (核心对象 / 体验 / 机制 / 人物).

## 多样性 check (Test 1 学到)

5 个搜索词应**至少覆盖 3 个维度**:
- 求职类: 技术 / 运营 / 产品 / 数据 / 设计 / 内容 — 选 3 方向
- 工具评测类: 上手 / 对比 / 缺点 / 用法 — 选 3 角度
- 人生方法论类: 年龄 / 学位 / 性别 / 场景 — 选 3 切片
